# quant-trading-system 快速上手

> 目标：从零开始跑起来，10 分钟内看到结果

---

## 一、环境准备

### 1.1 克隆代码

```bash
git clone git@github.com:jhZheng222/quant-trading-system.git
cd quant-trading-system
```

### 1.2 安装 Python

确保你用的是 **Python 3.10 或以上版本**（推荐 3.11）：

```bash
python3 --version
# Python 3.11.x ✅
```

> 💡 如果还没装 Python，推荐安装 [Miniconda](https://docs.conda.io/en/latest/miniconda.html)，
> 装完后 `conda create -n trading python=3.11` 创建一个新环境。

### 1.3 安装依赖

```bash
# 推荐用 venv 创建独立环境
python3 -m venv venv
source venv/bin/activate    # Mac/Linux
# venv\Scripts\activate     # Windows

# 安装依赖
pip install -r requirements.txt
```

> ⚠️ 如果 `ta-lib` 安装失败：Mac 用 `brew install ta-lib`，Linux 用 `apt install libta-lib-dev`

---

## 二、配置

### 2.1 复制配置文件

```bash
cp .env.example .env
```

### 2.2 填写 API Key（选做）

> **只想体验功能**：直接跳过这步，用模拟盘模式不需要 API Key。

如果想跑实盘，编辑 `.env` 文件：

```ini
# OKX 交易所配置（推荐）
OKX_API_KEY=你的API_KEY
OKX_SECRET_KEY=你的SECRET_KEY
OKX_PASSPHRASE=你的PASSPHRASE

# 或 Gate.io 配置
GATE_API_KEY=你的API_KEY
GATE_SECRET_KEY=你的SECRET_KEY
GATE_SANDBOX=true

# 初始资金（合约模拟盘用）
INITIAL_CAPITAL=10
LEVERAGE=20
```

> 🔑 如何获取 API Key？
> - OKX: 官网 → 个人中心 → API → 创建 API Key（交易权限即可）
> - Gate.io: 官网 → 账户设置 → API 管理 → 创建 Key

## 三、运行

### 3.1 查看系统状态

```bash
python main.py status
```

输出示例：
```
📊 量化交易系统状态
========================================
  ⚪ 无运行中的交易引擎
  根目录: /Users/xxx/quant-trading-system
  .env: ✅ 已配置
  数据库: trading.db (3960 KB)
========================================
```

### 3.2 运行回测（快速体验）

```bash
# 默认利弗莫尔策略 + DOGEUSDT
python main.py backtest

# 指定策略和交易对
python main.py backtest --strategy livermore --symbol DOGEUSDT

# 其他策略
python main.py backtest --strategy bollinger    # 布林带策略
python main.py backtest --strategy ema_rsi      # EMA+RSI 策略
python main.py backtest --strategy macd         # MACD 策略
```

### 3.3 模拟盘交易（推荐新手）

```bash
# 启动模拟盘（默认模式）
python main.py start
```

此时系统开始运行：
1. 连接 OKX 数据源（只读，不下单）
2. 策略分析行情
3. 在内存中模拟开平仓
4. 输出交易日志

按 `Ctrl+C` 停止。

### 3.4 实盘交易（有风险！）

```bash
python main.py start --live
```

> ⚠️ 实盘会真的下订单！建议先用模拟盘跑一周以上，
> 确认策略能稳定盈利再开实盘。

## 六、常见问题

### Q: 提示 `模块不存在`
```bash
ModuleNotFoundError: No module named 'xxx'

# 解决方法
pip install xxx
# 或者
pip install -r requirements.txt  # 一次性安装所有依赖
```

### Q: 跑不起来
1. 先执行 `python main.py status` 检查环境
2. 确认 `.env` 文件存在（可以留空跑模拟盘）
3. 看日志 `logs/trading_*.log` 定位错误
4. 检查网络能访问 OKX（国内网络需要科学上网）

### Q: 不想让我亏钱
放心！默认是**模拟盘**模式，只有加了 `--live` 才会下真实订单。
跑回测完全用历史数据，不涉及真金白银。

### Q: 哪里看详细的策略说明？
看 `docs/` 目录下的文档：
- `system_architecture.md` — 系统架构
- `data_source_architecture.md` — 数据源架构
- `websocket_service.md` — WebSocket 实现说明

### Q: 如何参与开发？
1. 读 `00-项目文档/quant-trading-system/重构计划.md` 了解整体规划
2. 看 `core/strategy/base_types.py` — 所有策略继承的基类
3. 在 `core/strategy/` 下加新策略文件，实现 `analyze()` 方法
4. 写测试 `tests/unit/test_*.py`
5. 提交 PR 🎉
